
**Requirements:**
- ✅ NO authentication required (public endpoint)
- ✅ Increment `views` by 1
- ✅ Return success status
- ✅ Handle duplicate views (optional: IP-based rate limiting)

**Example Implementation:**

```javascript
// POST /api/public/articles/:id/view
router.post('/public/articles/:id/view', async (req, res) => {
  const { id } = req.params;

  try {
    // Increment view count
    await db.query(
      'UPDATE articles SET views = views + 1 WHERE id = $1 AND status = $2',
      [id, 'aktif']
    );

    res.json({ success: true });
  } catch (error) {
    console.error('Error incrementing view:', error);
    res.status(500).json({ success: false, message: 'Failed to increment view' });
  }
});
```

## 📊 Optional Enhancements (Backend)

### 1. **IP-based Rate Limiting**

Prevent spam/abuse by limiting views per IP:

```javascript
const redis = require('redis');
const client = redis.createClient();

async function canIncrementView(articleId, ip) {
  const key = `view:${articleId}:${ip}`;
  const exists = await client.exists(key);

  if (exists) {
    return false; // Already viewed recently
  }

  // Set expiry to 1 hour (3600 seconds)
  await client.setex(key, 3600, '1');
  return true;
}
```

### 2. **Analytics Tracking**

Store view history for analytics:

```sql
CREATE TABLE article_views (
  id SERIAL PRIMARY KEY,
  article_id UUID REFERENCES articles(id),
  viewer_ip VARCHAR(45),
  viewer_country VARCHAR(2),
  viewed_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_article_views_article ON article_views(article_id);
CREATE INDEX idx_article_views_date ON article_views(viewed_at DESC);
```

### 3. **Trending Articles Query**

Return most viewed articles:

```sql
-- Most viewed in last 7 days
SELECT a.*, COUNT(av.id) as recent_views
FROM articles a
LEFT JOIN article_views av ON a.id = av.article_id
  AND av.viewed_at > NOW() - INTERVAL '7 days'
WHERE a.status = 'aktif'
GROUP BY a.id
ORDER BY recent_views DESC
LIMIT 10;
```
